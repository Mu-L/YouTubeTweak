你需要扫描项目中的 `src/assets/i18n/*.json` 文件，查找所有值中包含 `__NEED_TRANSLATE__` 标记的翻译条目，并完成这些条目的翻译。

## 母版语言

母版文件为：

```text
src/assets/i18n/zh-CN.json
```

所有翻译必须以 `zh-CN.json` 中相同 key 路径对应的中文内容为基准。

## 需要处理的文件

处理范围：

```text
src/assets/i18n/*.json
```

要求：

- 跳过 `src/assets/i18n/zh-CN.json`
- 只处理包含 `__NEED_TRANSLATE__` 标记的条目
- 不要重写整个文件
- 不要改动没有 `__NEED_TRANSLATE__` 标记的已有翻译
- 不要修改 JSON key、层级结构、排序和无关格式

## 上下文读取要求

在翻译前，必须读取以下目录中的前端页面文件：

```text
src/entrypoints/popup/pages
```

根据页面代码判断每个翻译 key 的实际使用位置、功能含义和 UI 场景，再进行翻译。

重点判断：

- 该文案是按钮、标题、配置项、提示语、说明文字，还是错误提示
- 是否属于播放器、字幕、截图、倍速、画面缩放、自定义 CSS、配置导入导出等功能
- 文案在 UI 中是否需要短句、动词、名词或说明型表达
- 是否存在容易误译的功能术语

## 翻译规则

翻译时必须遵守：

- 保持原文语义准确，不要自由发挥
- 保持 UI 文案简洁自然
- 保持同类功能术语一致
- 保留所有变量占位符，例如 `{name}`、`${value}`、`%s`
- 保留所有 HTML 标签、Markdown 标记、换行符和特殊符号
- 保留所有快捷键、单位、数字、版本号、配置值
- 不要翻译代码标识符、配置字段名、CSS 属性名、文件路径、URL
- 不要添加原文不存在的解释
- 不要删除任何已有 key

## 标记处理规则

如果某个值是：

```json
"xxx": "__NEED_TRANSLATE__中文内容"
```

则将其替换为目标语言翻译后的内容，并移除 `__NEED_TRANSLATE__` 标记。

如果某个值中间包含该标记，也只移除标记并完成对应内容翻译。

## 不确定时的处理

如果无法确定某个条目的含义：

1. 先根据 `src/entrypoints/popup/pages` 中的使用位置推断
2. 再参考同文件内已有翻译风格
3. 再参考其他语言文件中相同或相近 key 的表达
4. 仍无法确定时，保留 `__NEED_TRANSLATE__` 标记，并在最终输出中列出该 key 和原因

不要瞎翻。

## 完成后检查

完成翻译后必须检查：

- 所有被处理的 JSON 文件语法有效
- 所有已确认翻译的条目不再包含 `__NEED_TRANSLATE__`
- 未处理的 `__NEED_TRANSLATE__` 条目必须在最终报告中列出
- 没有破坏占位符、HTML 标签、Markdown、换行符
- 没有误改 `zh-CN.json`
- 没有改动无关条目

## 最终输出

任务完成后输出：

```markdown

### 已处理文件

- `文件路径`
- `文件路径`

### 已翻译条目

- `语言文件`: 数量

### 保留未翻译条目

如果没有，写：

无

如果有，按下面格式列出：

- `文件路径`
    - `key.path`: 原因

### 检查结果

- JSON 语法：通过 / 失败
- 占位符检查：通过 / 失败
- `__NEED_TRANSLATE__` 残留：无 / 有
```
