## 核心目标

根据项目真实代码逻辑，整理出扩展的全部功能，并按照扩展配置页面的结构进行分类。

最终文档结构必须尽量贴近 popup 配置面板中的分类顺序，格式为：

`大分类 -> 小分类 -> 功能`

例如：

```

# 播放器

这一类功能用于增强视频播放页面的使用体验。

## 锁定画质

### 自动切换视频画质

说明这个功能是什么、开启后会做什么、适合什么场景、有什么限制或注意事项。

```

## 分析范围

必须重点阅读并结合以下目录和文件：

1. `src/entrypoints/popup/pages`
    - 这是扩展的配置页面。
    - 需要根据这里的页面结构、分组、开关、选项，确定文档的大分类、小分类和功能顺序。

2. `src/inject/plugins`
    - 这是各个功能的主要实现位置。
    - 需要根据真实逻辑理解功能实际做了什么。

3. `src/inject/plugins_before`
    - 这是部分提前注入功能的实现位置。
    - 需要一并分析，不能遗漏。

4. `src/defaultConfig.ts`
    - 这是扩展的全部默认配置项。
    - 需要用它校对所有功能、开关、子选项、默认行为，避免遗漏隐藏配置。

除了以上重点位置，也需要扫描项目内其他相关代码，尤其是：

- 配置项被读取的位置
- 功能开关判断逻辑
- 功能之间的依赖关系
- 只在特定页面生效的逻辑
- 默认值、限制条件、特殊行为

## 文档要求

`FUNCTIONS.md` 必须满足以下要求：

1. 必须覆盖项目中所有用户可感知的功能。
2. 必须按照 popup 面板的分类结构组织内容。
3. 必须使用通俗、直接、易懂的中文。
4. 不要写代码。
5. 不要出现函数名、变量名、类名、组件名、实现文件路径等开发者内容。
6. 不要解释技术实现原理。
7. 不要使用晦涩术语。无法避免时，要用普通话解释清楚。
8. 每个功能都要说明：
    - 这个功能是什么
    - 开启后会发生什么
    - 用户什么时候会用到它
    - 有哪些可配置选项
    - 相关选项之间是否有依赖
    - 是否只在特定页面生效
    - 是否有注意事项或限制
9. 如果某个功能包含多个子选项，必须分别说明每个子选项的作用。
10. 如果某个功能需要另一个开关开启后才生效，必须明确写出依赖关系。
11. 如果某个功能会影响 YouTube 的不同页面，例如首页、搜索页、播放页、频道页、Shorts 页面、播放列表页面等，必须写清楚生效范围。
12. 如果某个配置项只影响显示效果、交互方式、快捷操作、默认行为，也必须单独说明。
13. 不要编造代码中不存在的功能。
14. 如果发现配置页面中有功能入口，但实现逻辑不完整，要用用户能理解的方式描述现有可见效果，不要写“代码未实现”这种开发者话术。
15. 最终只输出 `FUNCTIONS.md` 文件，不需要创建其他文档。

## 写作风格

文档要像产品说明书，不要像技术文档。

错误示例：

> 该功能通过监听 DOM 变化并调用播放器 API 修改 playbackRate。

正确示例：

> 开启后，扩展会在视频播放器中显示倍速控制入口，用户可以直接用鼠标切换播放速度，不需要打开 YouTube 原本的设置菜单。

错误示例：

> 读取 config.enableSpeedSlider 后初始化 SpeedSliderPlugin。

正确示例：

> 开启后，可以通过拖动倍速控制条来调整播放速度，也可以把鼠标放在倍速区域上滚动滚轮快速调整。

## 标题 ID 要求

`FUNCTIONS.md` 中所有标题都必须添加稳定的 URL 定位 ID，方便在 GitHub 中通过链接直接跳转到对应位置。

这里的“所有标题”包括：

- `# 一级标题`
- `## 二级标题`
- `### 三级标题`
- `#### 四级标题`
- 以及文档中出现的其他所有 Markdown 标题

标题 ID 使用 GitHub 支持的自定义锚点格式，写在标题上一行：

```markdown
<a name="functions"></a>

# 功能说明

<a name="player"></a>

# 播放器

<a name="player-quality"></a>

## 锁定画质

<a name="player-quality-auto-switch"></a>

### 自动切换视频画质
```

ID 命名规则：

1. 只能使用小写英文、数字和短横线。
2. 不要使用中文。
3. 不要使用空格。
4. 不要使用下划线。
5. 不要使用特殊符号。
6. ID 必须尽量简短，但要能看出含义。
7. 同一个文档中不能出现重复 ID。
8. 如果标题属于某个分类，ID 应体现层级关系。

推荐命名方式：

```markdown
<a name="player"></a>

# 播放器

<a name="player-quality"></a>

## 锁定画质

<a name="player-quality-auto-switch"></a>

### 自动切换视频画质

## 倍速按钮

<a name="player-speed-button-show"></a>

### 显示倍速按钮/倍速滑块
```

固定要求：

```markdown
<a name="functions"></a>

# 功能说明
```

文档开头的 `# 功能说明` 必须使用 `functions` 作为定位 ID。

如果存在同名标题，不能复用同一个 ID，必须根据所属分类生成不同 ID。

例如：

```markdown

```
